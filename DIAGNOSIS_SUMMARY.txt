PROBLEMA #1: DASHBOARD FINANCEIRO NÃO ATUALIZA EM TEMPO REAL
================================================================================

SINTOMA:
  - Usuário adiciona novo carro com preço R$ 100K
  - Dashboard mostra "Custos Gerais" = R$ 0 (não muda)
  - P&L não atualiza
  - Só atualiza após F5 (refresh manual)

CAUSA RAIZ:
  - useEffect tem dependência vazia [] → executa APENAS no mount
  - Sem polling automático ou WebSocket
  - Sem invalidação de cache após criar/atualizar veículo

ARQUIVO AFETADO:
  /src/frontend/pages/DashboardFinancial.jsx (linhas 13-59)

SEVERIDADE: CRÍTICO
IMPACTO: Usuário toma decisões financeiras com dados até 5 min atrasados

SOLUÇÃO ESTIMADA:
  - Implementar React Query com auto-refetch
  - Tempo: 1-2 horas

================================================================================
PROBLEMA #2: CRIAR/DELETAR GASTOS DÁ ERRO
================================================================================

SINTOMA:
  - Clica em "+ Nova Despesa" → erro genérico
  - Tenta deletar gasto → error, mas às vezes desaparece e volta ao F5
  - Tenta editar gasto → error vago
  - Mensagem: "Erro ao adicionar despesa" (sem detalhes)

CAUSA RAIZ:
  - Backend não valida se req.user.dealership_id existe
  - Erros PostgreSQL retornam 500 genérico, não erro específico
  - Falta logging detalhado
  - Validação de tipo de dados inadequada (string vs number, date format)

ARQUIVOS AFETADOS:
  /src/routes/expenses.js (linhas 33-65, 106-136, 138-159)
  /src/frontend/App.jsx (linhas 1490, 1507)

SEVERIDADE: CRÍTICO
IMPACTO: Módulo Financeiro completamente não-funcional
         Usuário não consegue registrar despesas → auditoria impossível
         P&L incompleto e inconsistente

SOLUÇÃO ESTIMADA:
  - Adicionar validações explícitas (dealership_id, tipos, format)
  - Melhorar error messages (detalhes técnicos)
  - Implementar retry logic no frontend
  - Tempo: 2-4 horas

================================================================================
PROBLEMA #3: CARRO VENDIDO/RESERVADO DESAPARECE APÓS F5
================================================================================

SINTOMA:
  - Seleciona carro no estoque
  - Clica "Marcar como Vendido"
  - Status muda na hora (visual atualiza)
  - Recarrega página (F5)
  - Carro desaparece da lista ou volta ao status anterior

CAUSA RAIZ:
  - Coluna sold_date não inicializa corretamente
  - Query de listagem sem índice composto (dealership_id, status)
  - PostgreSQL pode retornar dados inconsistentes com table scan lento

ARQUIVO AFETADO:
  /src/routes/inventory.js (linhas 522-530, 240-261)

SEVERIDADE: ALTO
IMPACTO: Double-booking de venda
         Dois vendedores tentam vender mesmo carro
         Operações de estoque confusas

SOLUÇÃO ESTIMADA:
  - Adicionar índice: (dealership_id, status)
  - Validar sold_date inicialização
  - Forçar TIMESTAMPTZ consistente
  - Tempo: 1-2 horas

================================================================================
PROBLEMA #4: EDIÇÕES DÃO ERRO (CARRO/CLIENTE/GASTO)
================================================================================

SINTOMA:
  - Editar informação de carro → "Erro ao atualizar veículo"
  - Editar cliente → erro
  - Editar gasto → erro
  - Mensagens vagas sem detalhe técnico

CAUSA RAIZ:
  - Sem validação de tipo de dados PRÉ-envio
  - Backend rejeita valores inválidos com erro 500 genérico
  - Sem sanitização de strings (XSS risk)
  - Sem validação de negócio (ex: purchasePrice > salePrice)

ARQUIVOS AFETADOS:
  /src/routes/inventory.js (linhas 502-580)
  /src/frontend/App.jsx (linhas 681, 657-701)

SEVERIDADE: ALTO
IMPACTO: Usuário não consegue corrigir dados
         Sempre cria novo ao invés de editar
         Duplicação de registros

SOLUÇÃO ESTIMADA:
  - Adicionar validações frontend (antes de enviar)
  - Adicionar validações backend (type, business logic)
  - Melhorar error messages
  - Tempo: 2-3 horas
