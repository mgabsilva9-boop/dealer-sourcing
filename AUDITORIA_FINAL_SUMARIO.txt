BUGS CRÍTICOS — DETALHES TÉCNICOS
================================================================================

BUG #1: GASTOS NÃO SALVAM
─────────────────────────
Arquivo: /src/routes/expenses.js (linhas 33-65)
Sintoma: POST /expenses/create → erro 500

Root Causes:
  1. Linha 48: req.user.dealership_id pode ser undefined
  2. Sem validação de amount (string vs number)
  3. Sem validação de date format
  4. Erro 500 genérico sem detalhes

Impacto: Módulo Financeiro completamente não-funcional
Severidade: CRÍTICO (5/5)
Estimativa de Fix: 1-2h

─────────────────────────────────────────────────────────────────────────────

BUG #2: DASHBOARD NÃO ATUALIZA
──────────────────────────────
Arquivo: /src/frontend/pages/DashboardFinancial.jsx (linhas 13-59)
Sintoma: Adiciona veículo → Dashboard continua mostrando totais antigos até F5

Root Cause:
  useEffect(() => { loadData(); }, []);
  ❌ Executa APENAS no mount, sem polling automático

Impacto: Gerente toma decisão com dados até 5 min atrasados
Severidade: CRÍTICO (5/5)
Estimativa de Fix: 2-3h

─────────────────────────────────────────────────────────────────────────────

BUG #3: CARRO VENDIDO DESAPARECE AO F5
────────────────────────────────────────
Arquivo: /src/routes/inventory.js (linhas 502-530, 240-261)
Sintoma: Marca como "vendido" → F5 → carro desaparece ou volta anterior

Root Causes:
  1. sold_date é DATE (sem timezone), não TIMESTAMP
  2. Query sem WHERE status filter (filtra no cliente)
  3. Sem índice composto (dealership_id, status) → table scan

Impacto: Dois vendedores vendem o mesmo carro (double-booking)
Severidade: ALTO (4/5)
Estimativa de Fix: 1-2h

─────────────────────────────────────────────────────────────────────────────

BUG #4: EDIÇÕES DÃO ERRO
────────────────────────
Arquivo: /src/routes/inventory.js (linhas 502-580)
Sintoma: PUT /inventory/:id → erro 500 ao editar preço, ano, etc

Root Causes:
  1. Sem validação de tipo (year: string vs number)
  2. Sem validação de business logic (purchasePrice > salePrice)
  3. Sem sanitização de strings (XSS risk)
  4. Erro 500 genérico ao rejeitar

Impacto: Usuário desiste de editar, cria novo → duplicação
Severidade: ALTO (4/5)
Estimativa de Fix: 2-3h
