BUG #1: EXPENSES NÃO SALVAM
───────────────────────────────────────────────────────────────────────────────
Problema:    POST /expenses/create retorna erro genérico 500
Root Cause:  Erro de constraint violation (23505) não diferenciado
Solução:     Retornar 409 para duplicatas, incluir hint para debug
Arquivo:     src/routes/expenses.js (linhas 94-105)
Mudanças:    +12 linhas
Resultado:   Erros específicos com hints para frontend
Teste:       curl -X POST /expenses/create (ver VERIFICATION_TESTS.md)

BUG #2: FINANCIAL TAB NÃO SINCRONIZA
───────────────────────────────────────────────────────────────────────────────
Problema:    Financeiro não atualiza quando vehicles mudam de status
Root Cause:  plData fetched 1x no mount, sem dependency em vehicles
Solução:     Substituir por useMemo que recalcula quando vehicles/expenses mudam
Arquivo:     src/frontend/App.jsx (linhas 1, 521, 623, 759-786)
Mudanças:    +1 import (useMemo) -1 useState -1 useEffect section +23 linhas
Resultado:   Dados sempre sincronizados, sem API overhead
Teste:       Marcar vehicle como 'sold' → números atualizam imediatamente

BUG #3: VEHICLE SOLD DESAPARECE
───────────────────────────────────────────────────────────────────────────────
Problema:    Veículo desaparece ou reapareça após marcar como 'sold'
Root Cause:  CURRENT_DATE (DATE) em coluna TIMESTAMPTZ causa conversão implícita
Solução:     Usar CURRENT_TIMESTAMP ao invés de CURRENT_DATE
Arquivo:     src/routes/inventory.js (linha 608)
Mudanças:    -1 linha (CURRENT_DATE → CURRENT_TIMESTAMP)
Resultado:   sold_date com tipo correto, dados persistem
Index:       idx_inventory_dealership_status já existe (migration 003)
Teste:       Marcar como sold → F5 → ainda está sold
