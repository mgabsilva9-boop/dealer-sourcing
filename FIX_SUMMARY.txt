PROBLEMA
─────────────────────────────────────────────────────────────────────────
Erro:
  - Failed to load resource: the server responded with a status of 405
  - URL: /null/auth/login ← ANORMAL (começa com /null/)
  - SyntaxError: Unexpected token '<' (HTML em vez de JSON)

Causa Raiz:
  1. api.js usava process.env.VITE_API_URL (❌ errado para client-side)
  2. Vercel Dashboard não tinha VITE_API_URL configurada
  3. Resultado: API_BASE = undefined | null = null
  4. Fetch para: `/null/auth/login` → 404/405 → HTML error → SyntaxError


SOLUÇÃO IMPLEMENTADA
─────────────────────────────────────────────────────────────────────────

1. ARQUIVO: src/frontend/api.js (linha 7)
   ANTES: const API_BASE = process.env.VITE_API_URL || ...
   DEPOIS: const API_BASE = import.meta.env.VITE_API_URL || ...
   MOTIVO: process.env não funciona em navegador, Vite usa import.meta.env

2. ARQUIVO: .env.production
   ADICIONADO:
   - VITE_API_URL=https://dealer-sourcing-api-production.up.railway.app
   - VITE_API_TIMEOUT=10000
   - VITE_LOG_LEVEL=error

3. ARQUIVO: .env.development (já existia, apenas confirmado)
   - VITE_API_URL=http://localhost:3000

4. ARQUIVO: test/verify-api-config.js
   - Script que valida a configuração localmente
   - Executa: npm run verify:api

5. ARQUIVO: package.json
   - Adicionado: "verify:api": "node test/verify-api-config.js"
